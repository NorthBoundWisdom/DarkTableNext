# Phase 0 Evidence

This directory contains version-controlled technical evidence collected before
the first Ravo engine target.  It separates facts that can be reproduced from
unapproved product decisions.

- [Legacy baseline report](legacy-baseline-2026-07-21.md) records the actual
  host, commands, legacy unit results, and an investigative Windows build that
  is not part of the frozen 0.9 baseline. The report is historical evidence
  only: it does not authorise further old builds, runs, or `src` fixes.
- [Capability inventory](capability-inventory.md) maps the current legacy IOP
  registry into Phase 1 and later Ravo work without declaring the whole legacy
  registry supported. `../../tools/check_capability_inventory.py` verifies
  that its 76 registry rows and fixture flags still match frozen CMake and the
  manifest.
- [Use-case contract](use-case-contract.md) fixes the machine-verifiable
  headless outcomes and records the deferred desktop cases that the engine and
  services must support before desktop implementation begins.
- [Product decision register](product-decision-register.md) names the owner,
  release-boundary, and compatibility decisions that source code cannot make.
- `../../tests/fixtures/vertical_slice_plan.json` identifies the first RAW
  vertical-slice candidate without requiring a reproduced legacy run; verify
  it with `../../tools/check_vertical_slice_plan.py`.
- `../../tests/fixtures/legacy_manifest.json` is generated by
  `../../tools/freeze_legacy_manifest.py`.  It provides deterministic hashes
  for every source image, XMP, expected PNG, and CPU/GPU threshold currently
  tracked by `darktable-tests/`.
- `../../tests/fixtures/fixture_classification_ledger.json` gives every
  manifest fixture exactly one pre-acceptance disposition row. It currently
  records every fixture as `unclassified`; this prevents an unmentioned skip,
  but does not mean an input is missing or make a Ravo acceptance claim. Validate it with
  `../../tools/check_fixture_classification_ledger.py`.

The frozen 0.9 source and fixture reference is commit
`320970bf7c9cbbc6611cfc3eb60f8f2b0424b782` (2026-07-21, “Freeze 0.9 and
prepare the Ravo rewrite”). Its `src` tree is
`a3ac761ecbb0cf668ecad49aff8bd0e29235f5f7` and its `darktable-tests` tree is
`1dc38893f39e113620aebbbdc927218ca4a2b8af`; both match the current committed
HEAD. The same check also protects the frozen root CMake graph, `cmake/`,
`data/`, and `packaging/`; it deliberately leaves Ravo, FreeCM, dependency
workflow configuration, and benchmarks outside that legacy boundary. The
product owner and Ravo review owner remain governance decisions for product
scope. The old project, old CLI, old CTest, packaging targets, and
`darktable-tests/run` are frozen and must not be configured, compiled, or
executed. The committed fixture manifest is the complete static input set;
later acceptance evidence is produced only by Ravo. That does not change the
accepted C++20, headless-first, and Ravo-only-growth ADRs.

Verify that the frozen fixture inventory is stable without writing it:

```powershell
python Ravo/tools/freeze_legacy_manifest.py --check
python Ravo/tools/check_capability_inventory.py
python Ravo/tools/check_vertical_slice_plan.py
python Ravo/tools/check_fixture_classification_ledger.py
python Ravo/tools/check_freeze_reference.py
```

The generator inventories committed reference material only. Frozen legacy
fixtures must not change as ordinary Ravo development. It never invokes an old
binary or test runner, and a matching manifest is not proof of an accepted
Ravo image result.

New Ravo tests use GoogleTest (and GoogleMock where an interaction needs to be
verified). CMocka remains confined to the legacy `src/tests` targets.
GoogleTest 1.17.0 is available through the existing Windows toolchain; the
Phase 1 Ravo targets and their actual validation commands are documented in
[Ravo/README.md](../../README.md#构建与测试).
