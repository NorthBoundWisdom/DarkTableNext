# Phase 0 Evidence

This directory contains version-controlled technical evidence collected before
the first Ravo engine target.  It separates facts that can be reproduced from
unapproved product decisions.

- [Legacy baseline report](legacy-baseline-2026-07-21.md) records the actual
  host, commands, legacy unit results, and an investigative Windows build that
  is not part of the frozen 0.9 baseline. It is not a green image-regression
  claim and does not authorise further `src` fixes.
- [Capability inventory](capability-inventory.md) maps the current legacy IOP
  registry into Phase 1 and later Ravo work without declaring the whole legacy
  registry supported.
- `../../tests/fixtures/legacy_manifest.json` is generated by
  `../../tools/freeze_legacy_manifest.py`.  It provides deterministic hashes
  for every source image, XMP, expected PNG, and CPU/GPU threshold currently
  tracked by `darktable-tests/`.

The product owner, exact 0.9 frozen commit/artifact, and Ravo review owner are
not discoverable from source control. They remain governance decisions for the
maintainers and are intentionally not filled with invented values here. That
prevents a formal Phase 0 exit, but does not change the accepted C++20,
headless-first, and Ravo-only-growth ADRs.

Re-run the generator to verify that the frozen fixture inventory is stable:

```powershell
python Ravo/tools/freeze_legacy_manifest.py
git diff -- Ravo/tests/fixtures/legacy_manifest.json
```

The script inventories committed reference material only. Frozen legacy
fixtures must not change as ordinary Ravo development. The script must never
run `darktable-tests/run --update-expected`, and a regenerated hash is not
proof of an accepted Ravo image result.

New Ravo tests will use GoogleTest (and GoogleMock where an interaction needs
to be verified). CMocka remains confined to the legacy `src/tests` targets.
GoogleTest 1.17.0 is available in this Windows workspace; no Ravo CMake target
exists yet, so this is preparation evidence rather than a test result.
